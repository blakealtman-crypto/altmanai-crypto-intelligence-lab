# Contributing

Thanks for considering a contribution to AltmanAI Crypto Intelligence Lab.

## Ground Rules

1. Read `DISCLAIMER.md` and `docs/COMPLIANCE_NOTES.md` first — all contributions must stay within the educational/research scope. No trading signals, price predictions, or advice-like content.
2. Keep claims accurate and sourced where reasonable. No fabricated statistics or unverifiable claims.
3. Prefer clear, plain-English explanations over jargon.

## How to Contribute

1. Open an issue first for anything non-trivial (see `.github/ISSUE_TEMPLATE/`) so we can align before you invest time.
2. Fork the repo and create a branch: `feature/your-change` or `docs/your-change`.
3. Make your changes with clear commit messages.
4. Open a pull request using the PR template — describe what changed and why.

## Code Style

- TypeScript preferred throughout `src/`, `components/`, and `data/`.
- Keep components small and focused; prefer composition over large monolithic components.

## Research Content Contributions

Use the `research_note` issue template to propose new glossary terms or research topics — include your sources.
